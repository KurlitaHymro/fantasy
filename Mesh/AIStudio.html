1.1.3 骨骼

事先说一下,本文中的骨骼指的是骨骼动画里的骨骼.简单来说就是给你一个3d模型,它是由面组成的,为了方便它摆姿势,我们不可能对模型的每个顶点一一移动位置.我们知道,生物的动作其实是肌肉拉动骨骼进行旋转完成的.在动画制作中,动画骨骼也是类似于生物骨骼的东西,可以理解为由数量较少的线段构成;我们将模型的顶点按照不同的权重与动画骨骼绑定,当我们旋转移动骨骼时,与之绑定的顶点也会发生相应的位置变换,从而形成模型的姿势改变.

在教程中,使用的模型是从网上下载的人体骨架的3d模型,为了防止混淆,在本文中称为"骷髅";

而"骨骼"则特指骨骼动画的骨骼.

1.1.4 FK,IK

FK正向动力学，请参考芭比娃娃的运动方式

IK反向动力学，请参考提线木偶的运动方式

芭比娃娃是靠活动各个关节来带动下个关节的运动，比如说你旋转肩膀，会带动胳膊运动

提线木偶是靠小关节的运动，而使得上面关节跟着运动，比如说你拉一下提线木偶的手部关节，会带动胳膊和肩膀一起运动

没有“什么时候用IK什么时候用FK”这么一说，因为人的运动一直都是FK和IK一起的作用。只要适合的场合用适合的动作就OK。

引自:https://zhidao.baidu.com/question/110064337.html

studio的男角色和女角色支持FK和IK,而物品的话,目前我只找到FK的解决方案,貌似并不支持IK.

1.3 任务明确

再次提醒,本教程的内容是:用 Blender 对网络上下载的 .blend 模型进行绑骨,刷权重,然后将模型导出,并用SB3Utility导入AI少女Studio,作为 FK 物品.最终我们将获得一个可以摆姿势的骷髅.(附带变色和发光功能)

如果你想做的东西不是这个,本教程可以作为参考,遇到问题欢迎来找我交流.

一个zipmod形式的AI studio FK 物品由以下部分组成:

abdata # 文件夹
    studio # 文件夹
        GroupName # 文件夹
            mod_name.unity3d # 3d资源文件
        info # 文件夹
            GroupName # 文件夹
                ItemBoneList_Group_Category.csv # 物品的骨骼列表
                ItemCategory_Category_Group.csv # 物品的子类别
                ItemGroup_GroupName.csv # 物品的组别
                ItemList_ListID_Group_Category.csv # 物品清单
manifest.xml # 用于制作manifest

其中.csv是表格文件,属于mod中的"list"部分,用来给SideLoader插件生成一个包含表格信息的.unity3d文件,这个文件写好以后,在游戏里的相应列表里看到相应的选项;.unity3d文件是资源文件,存放我们的模型;manifest.xml用来给SideLoader插件生成manifest文件,manifest文件是list和资源文件的总索引,貌似还有加密的功能,这个文件不是很懂,照着格式填就行了.

我发现,现在胳膊和手的位置不太好,会给之后绑骨增加麻烦,我们调整一下位置和角度:

  1. Shift+鼠标右键设定3D游标的位置
  2. 将轴心点设置为3D游标
  3. 根据需要选择和调节object的位置和角度

调好位置与角度以后,Shift+s,游标->世界中心;轴心点调回各自的原点.

3 用 SB3Utility 制作 MOD 的3d资源部分

我们需要一个.fbx格式的模型,将它导入SB3Utility,适当编辑以后,做成MOD.

3.1 模型导出前的处理

把骨骼的小眼睛勾掉,防止对我们造成干扰.

3.1.1 四边面->三角面

游戏中的3d资源一般都是按照三角面的形式来保存的,而建模的时候一般则是四边面建模,所以我们需要把四边面转化为三角面.

物体模式下,选中object,然后切换到编辑模式,再a全选.选择模式切换为面,然后打开面这个菜单,选面三角化.

留意右下角的资源统计,面(三角,四边,五边等等)和三角的数量相等了,说明现在所有的面都是三角面.

3.1.2 重算法线

在之前的Shift+d和x轴向-1倍拉伸以后,得到的镜像复制品的法线反向可能都指向模型内部了,这样是不行的.

法线方向你可以简单理解为一个平面的朝向.illusion在渲染模型的时候会用到法线方向.如果你是发光物体,法线向内就会朝内发光,而外面看来就是黑咕隆咚的,仿佛黑夜里没有窗户的屋子,屋内确实亮堂堂的.法线向外的话,渲染以后在游戏里它就会对外发光.法线方向还会影响材质,反射等等各种表面属性的渲染,所以不容忽视.

  • 什么是渲染?

    简单来说就是你把模型的顶点坐标,表面材质之类的参数传给电脑,电脑根据这些数据,计算出要显示在显示屏上的光影效果.

  • 怎么重算法线呢?

    a全选所有的面,然后Shift+n就行了.

  • 怎么检验法线方向?

    选择你要检验的面,坐标系选为法向,把物体Gizmo-移动勾选上.z轴正方向就是法线方向.

主要检查一下之前-1倍缩放得到的面的法线方向,如左臂.

还有一种检查方法是利用"沿法向挤出"看看挤出的面的方向.请自行上网查阅资料.使用这一方法要注意右下角的资源统计信息,因为即使你右键放弃了挤出,顶点和面还是被复制了,要记得x删除.否则挤出几次你的内存就爆掉了.Shift+d与此类似,也是创建一个副本,即使你不移动,也会在原地有重复顶点,要谨慎使用.

由于我们是要将外部的3d资源导入游戏内部,如果不给游戏提供一个UV展开方式,那么游戏中这个物体表面的材质和贴图就是乱的.

在本教程中,我们没有制作贴图,我们只需要展开一下UV,保存一下,让游戏的渲染器明白模型表面的展开方式就行了.

  3. 右键-UV展开面-智能UV投射,在弹出的窗口点确定

  1. 左边的Tab里,选择图像绘制,就可以看到UV展开的样子.

Ctrl+s保存一下,此时模型已经有了UV展开方式的信息.我们换回Default排版.

其中Group,Category由你自己定义,比如我的Group是1218,Category是01.Group尽量大一点以免和游戏原来的Group冲突,但也不要太大以免超过最大限度(貌似7000还是可以的).Category作为你的Group里面的小种类,序号自己定义即可.

这几个文件的编写可以在附件里找我的案例,建议用Excel编辑.格式建议自己打开一个AI少女Studio相应的list,和官方的格式保持一致.

5 MOD压缩打包

在[Eirrac] Simple_Skeleton文件夹里,新建一个空白文档,填写如下内容(最好都用英文填写):

xxxxxxxxxx
<manifest schema-ver="1">
    <guid>Eirrac.Studio.Simple_Skeleton</guid>
    <name>Simple Skeleton</name>
    <version>1.0.0</version>
    <author>Eirrac</author>
    <description>
        This is a FK human skeleton in AI Girl Studio with color changing and Emission.
        You can find it as Items/Eirrac/FK/Simple Skeleton in AI Studio.
    </description>
    <website>zodgame.xyz</website>
    <game>AI Girl</game>
</manifest>

其中,<guid>这一行是必须有的,官方推荐格式为com.Author.Mod name,我这里的格式和推荐的不一样,无伤大雅,只要保证你的这个内容不会和其他任何人重复就行(带有相同guid的不同mod同时加载会出错),它是用来计算你的mod的guid的(记得之前我们改过skeleton.unity3d的CAB吗,其实可以不改,但是为了安全起见,改了更为保险).

其他的<name><version><author><description><website><game>都是可有可无.比如你也可以这么填:

xxxxxxxxxx
<manifest schema-ver="1">
    <guid>Eirrac.Studio.Simple_Skeleton</guid>
    <name></name>
</manifest>

改好以后,保存并关闭,把文件名改为manifest.xml.

现在你的zipmod已经接近完成了,文件结构如下(在1.3任务明确里面已经讲过,为了你读起来方便,我在下面复制了一遍):

xxxxxxxxxx
[Author] mod name # 文件夹
    abdata # 文件夹
        studio # 文件夹
            GroupName # 文件夹
                mod_name.unity3d # 3d资源文件
            info # 文件夹
                GroupName # 文件夹
                    ItemBoneList_Group_Category.csv # 物品的骨骼列表
                    ItemCategory_Category_Group.csv # 物品的子类别
                    ItemGroup_GroupName.csv # 物品的组别
                    ItemList_ListID_Group_Category.csv # 物品清单
    manifest.xml # 用于制作manifest

将abdata和manifest.xml打包,压缩方式选择为储存(我用的是2345好压).

将扩展名改为.zipmod防止不懂mod的人解压出来.

你制作的mod就是[Eirrac] Simple_Skeleton.zipmod.
